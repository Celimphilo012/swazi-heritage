Project Overview

The Swazi Cultural Heritage Platform is a full-stack web application designed to preserve, document, and share the cultural traditions of the Kingdom of Eswatini. It serves three distinct user roles — general users, cultural practitioners, and administrators — each with a dedicated dashboard.

LayerTechnologyPort
FrontendReact 18, Vite 5, Tailwind CSS 3, React Router 65173
BackendNode.js, Express 4, JWT Auth, Multer, Winston5000
DatabaseMySQL 8 (via mysql2)3306
AI ChatGoogle Gemini API (with local NLP fallback)—
Cinema SchedulerBuilt-in Node.js interval (auto-starts sessions)—
ℹ️
Monorepo structure The project uses a single Git repository with two separate Node.js packages: client/ (Vite React app) and server/ (Express API). Each has its own package.json and must be installed independently.

Server Setup

Navigate to the server directory 
cd server
Install dependencies 
npm install
Create the environment file

Copy the example file and fill in your values:

# Windows
copy .env.example .env

# macOS / Linux
cp .env.example .env

Then open .env in your editor and set each value:

# ── Server ────────────────────────────
PORT=5000
NODE_ENV=development
CLIENT_URL=http://localhost:5173

# ── Database ──────────────────────────
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=swazi_heritage

# ── JWT (use long random strings) ─────
JWT_SECRET=change_this_to_a_long_random_secret
JWT_EXPIRES_IN=7d
JWT_REFRESH_SECRET=change_this_to_another_long_random_secret
JWT_REFRESH_EXPIRES_IN=30d

# ── Gemini AI ─────────────────────────
GEMINI_API_KEY=your_gemini_api_key

# ── Uploads ───────────────────────────
MAX_FILE_SIZE_MB=5
⚠️
Never commit your .env file It is already in .gitignore. Keep JWT secrets long and random. You can generate a random string with:
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
🔑
Getting a Gemini API Key (free) Go to aistudio.google.com → Sign in with Google → Click "Get API key" → Create API key. Paste the key into GEMINI_API_KEY. Without this key the server will refuse to start. The AI chat falls back to a local NLP model if Gemini is unavailable at runtime.

Database Setup & Migrations

The project ships with a single migration script that creates the database, applies the full schema, and seeds default data (admin account, imvunulo presets, and system config).

Ensure MySQL is running

Your MySQL user must have CREATE DATABASE privileges. The migration script will create the database automatically if it does not exist.

Run the migration from the server directory 
# Make sure you are inside server/
npm run migrate

Expected output:

Running migrations...
Schema applied.
Seeding...
  Admin created: admin@swaziheritage.sz / Admin@1234
  CHANGE THIS PASSWORD after first login!
  10 imvunulo presets seeded.
  3 config entries seeded.
Done.
Apply cinema status migration (if the DB was seeded before v1.1)

If the database was initialised before the recorded-video feature was added, run these two SQL statements in MySQL Workbench or the MySQL CLI:

ALTER TABLE cinemas
  MODIFY COLUMN status
  ENUM('scheduled','live','ended','cancelled','available','unavailable')
  NOT NULL DEFAULT 'scheduled';

-- Set existing recorded sessions to available
UPDATE cinemas SET status = 'available' WHERE type = 'recorded';
ℹ️
The npm run migrate command re-applies schema.sql which already contains the updated ENUM, so this manual step is only needed if you ran migrations on an older version of the project.

Running the Project

Both the server and client must run simultaneously. Open two separate terminal windows and run one process in each.

Terminal 1 — Start the API server

# From project root
cd server
npm run dev       # uses nodemon for auto-reload on file changes

# OR for a stable (non-reloading) process:
npm start
✅
Server is ready when you see: Server running in development mode on port 5000

Terminal 2 — Start the frontend

# From project root
cd client
npm run dev
✅
Frontend is ready when you see: Local: http://localhost:5173/

Open http://localhost:5173 in your browser.

🔄
Cinema session auto-scheduler The server runs a background task every 30 seconds that automatically transitions cinema sessions from scheduled → live once their scheduled_at datetime is reached.

Default Credentials & Roles

The migration seed creates one admin account. All other accounts are registered through the platform.

Email
admin@swaziheritage.sz
Password
Admin@1234
Change after first login

User Roles

RoleAccessHow to assign
user Browse ceremonies & lineage, use AI chat, book & watch cinema sessions Default on self-registration
history_keeper Practitioner dashboard — create/edit lineage records and clans Admin upgrades via User Management
ceremony_keeper Practitioner dashboard — create/edit ceremonies and songs Admin upgrades via User Management
admin Full access — users, content review, cinema, config, analytics, audit log, ML model Only via direct DB or by another admin
⚠️
Content review workflow All ceremony and lineage records submitted by practitioners start with status pending_review. An admin must approve (publish) or reject them from the Content Review page before they appear publicly.
10

Post-Setup Checklist

  • Log in as admin and immediately change the default password in Profile Settings
  • Verify the AI chat works at /chat — this confirms the Gemini API key is valid
  • Visit Admin → Config and review the AI system prompt and platform name
  • Create at least one history_keeper and one ceremony_keeper account to test the practitioner workflows
  • Confirm the server/uploads/ folder exists and is writable (the server creates it on first file upload)
  • Replace JWT secrets in .env with real random values (see Section 5 for the generation command)

Production Build

Build the frontend

cd client
npm run build     # outputs to client/dist/

The dist/ folder contains static files. Serve them with Nginx, Apache, or any static host. Configure your web server to proxy /api requests to the Node.js server.

Run the server in production

NODE_ENV=production node server/server.js

# Recommended: use PM2 for process management
npm install -g pm2
pm2 start server/server.js --name swazi-heritage-api
pm2 save
pm2 startup
⚠️
In production, set NODE_ENV=production, update CLIENT_URL to your live domain, and serve the frontend over HTTPS. Make sure the database credentials and JWT secrets are strong unique values.

Troubleshooting

SymptomCauseFix
ECONNREFUSED 3306 MySQL not running or wrong host/port in .env Start MySQL service; double-check DB_HOST and DB_PORT
Missing required variables: GEMINI_API_KEY .env file missing or key not filled in Create server/.env from .env.example and add your key
Access denied for user 'root' Wrong MySQL password Update DB_PASSWORD in server/.env
API calls return 404 in browser Server not running or wrong port Ensure npm run dev is running in server/ on port 5000
Vite proxy errors / CORS Server not on port 5000, or CLIENT_URL mismatch Confirm server runs on 5000; check CLIENT_URL=http://localhost:5173 in server/.env
Recorded videos not showing on cinema page Old DB schema missing available status in ENUM Run the ALTER TABLE command in Section 7, Step 3
File upload fails silently server/uploads/ folder missing or not writable Create the folder manually: mkdir server/uploads
npm run migrate fails with "ER_PARSE_ERROR" MySQL version < 8 or multipleStatements disabled Upgrade to MySQL 8. The migrate script enables multipleStatements automatically
AI chat returns "Fallback" responses only Invalid or expired Gemini API key Regenerate key at aistudio.google.com and update GEMINI_API_KEY
nodemon not found Dev dependencies not installed Run npm install inside server/
